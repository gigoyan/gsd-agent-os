# Target UI Behavior Model

## Purpose
Define intended UI behavior before implementation recommendations.

## Required State Dimensions
- default
- edit
- dirty/unsaved
- inactive/disabled/read-only
- destructive action
- empty
- loading
- error
- success
- permission/role-based
- responsive/mobile
- accessibility

## Rule
If state evidence is unavailable, mark it `Unknown` and list it as an evidence gap.
