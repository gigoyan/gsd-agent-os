# Root Cause Clustering

## Purpose
Prevent flat checklist findings.

## Workflow
1. List observed symptoms.
2. Group by likely root cause.
3. Mark root cause evidence status.
4. Report structural causes first.
5. Put local polish after structural risks.

## Structural UX Risks
- wrong interaction model
- missing state lifecycle
- unsafe destructive action
- poor hierarchy
- inaccessible custom pattern
- unclear surface ownership
- missing permission behavior
- evidence gap

## Local Polish
- spacing
- alignment
- minor copy
- minor color/token inconsistency
- minor visual density
